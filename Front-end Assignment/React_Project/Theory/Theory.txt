Q2: Explain the core principles of React such as the Virtual DOM and component-based architecture.
Answer:
Virtual DOM:

The Virtual DOM is a lightweight in-memory copy of the real DOM.
When state/props change, React first updates the Virtual DOM, then diffs it with the previous version, and finally updates only the changed parts in the real DOM — this process is called Reconciliation.
This makes React extremely fast compared to direct DOM manipulation.

User Action → State Change → Virtual DOM Updated → Diff with Old VDOM → Real DOM Patched
Component-Based Architecture:

The UI is broken into small, reusable, independent pieces called components.
Each component manages its own logic and UI.
Components can be nested, composed, and reused across the application.

jsx// Example of component composition
function App() {
  return (
    <div>
      <Header />
      <MainContent />
      <Footer />
    </div>
  );
}

Q3: Discuss the importance of using curly braces {} in JSX expressions.
Answer:
Curly braces {} act as a "JavaScript mode switch" inside JSX. They allow you to embed any valid JavaScript expression directly into your markup.
jsxconst user = { name: "Alice", score: 95 };
const isLoggedIn = true;
const items = ["Apple", "Banana", "Cherry"];

function App() {
  return (
    <div>
      {/* Variables */}
      <h1>{user.name}</h1>

      {/* Expressions */}
      <p>Score: {user.score * 2}</p>

      {/* Ternary */}
      <p>{isLoggedIn ? "Welcome back!" : "Please log in"}</p>

      {/* Function calls */}
      <p>{items.join(", ")}</p>

      {/* Array map */}
      <ul>
        {items.map((item, index) => (
          <li key={index}>{item}</li>
        ))}
      </ul>
    </div>
  );
}
Without {}, everything inside JSX is treated as plain text.

Q2: Explain the concept of state in React and how it is used to manage component data.
Answer:
State is a built-in React object that stores dynamic data belonging to a component. When state changes, React automatically re-renders the component.
jsximport { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);  // Initial state = 0

  return (
    <div>
      <p>Current Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <button onClick={() => setCount(count - 1)}>Decrement</button>
      <button onClick={() => setCount(0)}>Reset</button>
    </div>
  );
}

Q3: Why is this.setState() used in class components, and how does it work?
Answer:
this.setState() is used in class components to update state. You must never mutate this.state directly — always use setState().
jsxclass Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };  // Initial state
  }

  increment = () => {
    this.setState({ count: this.state.count + 1 });
    // React schedules a re-render after setState
  };

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={this.increment}>Increment</button>
      </div>
    );
  }
}
Why setState() and not direct assignment?

Direct mutation (this.state.count = 5) does NOT trigger a re-render
setState() tells React something changed → triggers re-render
setState() is asynchronous — React batches updates for performance

Q3: Why do you need to bind event handlers in class components?
Answer:
In JavaScript, class methods do not automatically bind this. If you don't bind, this will be undefined inside the handler.
jsxclass MyComponent extends React.Component {
  constructor(props) {
    super(props);
    this.state = { clicked: false };
    // Method 1: Bind in constructor
    this.handleClick = this.handleClick.bind(this);
  }

  handleClick() {
    this.setState({ clicked: true });  // 'this' works because of binding
  }

  render() {
    return <button onClick={this.handleClick}>Click</button>;
  }
}

// Modern alternative: Arrow functions (auto-bind 'this')
class MyComponent extends React.Component {
  handleClick = () => {
    this.setState({ clicked: true });  // Arrow function — no binding needed
  };

  render() {
    return <button onClick={this.handleClick}>Click</button>;
  }
}

7. Lists and Keys
Q1: How do you render a list of items in React? Why is it important to use keys when rendering lists?
Answer:
You render lists using JavaScript's .map() function inside JSX. Keys help React identify which items changed, were added, or removed — enabling efficient re-rendering.
Without keys, React re-renders the entire list. With unique keys, React updates only the changed items.

Q2: What are keys in React, and what happens if you do not provide a unique key?
Answer:
Keys are special string attributes you must include when rendering lists. They must be unique among siblings.
If you don't provide keys (or use non-unique keys):

React shows a warning in the console
Performance degrades because React can't efficiently diff the list
State/animation bugs can occur when items reorder

jsx// ❌ Bad — no key
items.map(item => <li>{item}</li>)

// ⚠️ Okay but not ideal — index as key (avoid if list can reorder)
items.map((item, index) => <li key={index}>{item}</li>)

// ✅ Best — unique ID as key
items.map(item => <li key={item.id}>{item.name}</li>)

9. Lifecycle Methods (Class Components)
Q1: What are lifecycle methods in React class components? Describe the phases of a component's lifecycle.
Answer:
Lifecycle methods are special methods that automatically get called at different stages of a component's life.
3 Phases:
MOUNTING          UPDATING           UNMOUNTING
─────────         ────────           ──────────
constructor()     render()           componentWillUnmount()
render()          componentDidUpdate()
componentDidMount()

Mounting — Component is created and inserted into the DOM
Updating — Component re-renders due to state/props change
Unmounting — Component is removed from the DOM


Q2: Explain the purpose of componentDidMount(), componentDidUpdate(), and componentWillUnmount().
Answer:
jsximport React, { Component } from 'react';

class LifecycleDemo extends Component {
  constructor(props) {
    super(props);
    this.state = { data: null, count: 0 };
  }

  // Called ONCE after component is inserted into DOM
  // Perfect for: API calls, subscriptions, DOM manipulation
  componentDidMount() {
    console.log("Component mounted!");
    fetch("https://jsonplaceholder.typicode.com/users/1")
      .then((res) => res.json())
      .then((data) => this.setState({ data }));
  }

  // Called after every re-render (when state/props change)
  // Perfect for: reacting to prop changes, updating DOM after render
  componentDidUpdate(prevProps, prevState) {
    if (prevState.count !== this.state.count) {
      console.log("Count updated to: " + this.state.count);
    }
  }

  // Called just before component is removed from DOM
  // Perfect for: cleanup — clear timers, cancel API calls, unsubscribe
  componentWillUnmount() {
    console.log("Component unmounted! Cleaning up...");
  }

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={() => this.setState({ count: this.state.count + 1 })}>
          Increment
        </button>
        {this.state.data && <p>User: {this.state.data.name}</p>}
      </div>
    );
  }
}

10. Hooks (useState, useEffect, useReducer, useMemo, useRef, useCallback)
Q1: What are React hooks? How do useState() and useEffect() hooks work in functional components?
Answer:
Hooks are special functions that let functional components use React features like state and lifecycle methods — previously only available in class components. Introduced in React 16.8.
jsximport { useState, useEffect } from 'react';

function HooksDemo() {
  // useState: manages local state
  const [count, setCount] = useState(0);
  const [data, setData] = useState(null);

  // useEffect: runs side effects after render
  // Equivalent to componentDidMount + componentDidUpdate + componentWillUnmount
  useEffect(() => {
    document.title = `Count: ${count}`;  // Runs after every render

    fetch("https://jsonplaceholder.typicode.com/users/1")
      .then(res => res.json())
      .then(data => setData(data));

    return () => {
      // Cleanup function (like componentWillUnmount)
      console.log("Cleanup!");
    };
  }, [count]);  // Dependency array — runs only when 'count' changes
                // [] = run only once (on mount)
                // No array = run after every render

  return <div><p>{count}</p><button onClick={() => setCount(count + 1)}>+</button></div>;
}

Q3: What is useReducer? How do we use it in a React app?
Answer:
useReducer is an alternative to useState for managing complex state logic. It follows the Redux pattern: dispatch an action → reducer returns new state.
jsximport { useReducer } from 'react';

// Reducer function: pure function that returns new state
function counterReducer(state, action) {
  switch (action.type) {
    case "INCREMENT": return { count: state.count + 1 };
    case "DECREMENT": return { count: state.count - 1 };
    case "RESET":     return { count: 0 };
    default:          return state;
  }
}

function Counter() {
  const [state, dispatch] = useReducer(counterReducer, { count: 0 });

  return (
    <div>
      <p>Count: {state.count}</p>
      <button onClick={() => dispatch({ type: "INCREMENT" })}>+</button>
      <button onClick={() => dispatch({ type: "DECREMENT" })}>-</button>
      <button onClick={() => dispatch({ type: "RESET" })}>Reset</button>
    </div>
  );
}
Use useReducer when state has multiple sub-values or complex transition logic.

Q4: What is the purpose of useCallback & useMemo Hooks?
Answer:
useMemo — Memoizes the result of a computation. Prevents expensive recalculations on every render.
jsximport { useMemo, useState } from 'react';

function App() {
  const [count, setCount] = useState(0);
  const [input, setInput] = useState("");

  // Only recomputes when 'count' changes, not when 'input' changes
  const expensiveValue = useMemo(() => {
    console.log("Computing...");
    return count * 1000;
  }, [count]);

  return (
    <div>
      <p>Result: {expensiveValue}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <input value={input} onChange={e => setInput(e.target.value)} />
    </div>
  );
}
useCallback — Memoizes a function reference. Prevents child re-renders when passing callbacks as props.
jsximport { useCallback, useState } from 'react';

function App() {
  const [count, setCount] = useState(0);

  // Same function reference between renders (unless count changes)
  const handleClick = useCallback(() => {
    setCount(c => c + 1);
  }, []);  // No dependencies — function never changes

  return <ChildComponent onClick={handleClick} />;
}

Q6: What is useRef? How to work with it in a React app?
Answer:
useRef returns a mutable ref object whose .current property persists for the full lifetime of the component. It does NOT trigger re-renders when changed.
Two main uses:
jsximport { useRef, useState, useEffect } from 'react';

function RefDemo() {
  // Use 1: Access DOM element directly
  const inputRef = useRef(null);

  function focusInput() {
    inputRef.current.focus();  // Direct DOM access
  }

  // Use 2: Store mutable value without causing re-render
  const renderCount = useRef(0);
  const [text, setText] = useState("");

  useEffect(() => {
    renderCount.current += 1;  // Doesn't trigger re-render
  });

  return (
    <div>
      <input ref={inputRef} value={text} onChange={e => setText(e.target.value)} />
      <button onClick={focusInput}>Focus Input</button>
      <p>Render count: {renderCount.current}</p>
    </div>
  );
}

Q5: Discuss the importance of handling errors and loading states when working with APIs in React.
Answer:
jsximport { useState, useEffect } from 'react';

function DataFetcher() {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);   // Show loading spinner
  const [error, setError] = useState(null);        // Show error message

  useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/users")
      .then(res => {
        if (!res.ok) throw new Error("Network response was not ok");
        return res.json();
      })
      .then(data => {
        setData(data);
        setLoading(false);
      })
      .catch(err => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  if (loading) return <p>⏳ Loading...</p>;
  if (error) return <p>❌ Error: {error}</p>;

  return (
    <ul>
      {data.map(user => <li key={user.id}>{user.name}</li>)}
    </ul>
  );
}
Why it matters:

Users know the app is working (loading state)
Prevents showing blank/broken UI (error state)
Improves user experience and trust


13. Context API
Q1: What is the Context API in React? How is it used to manage global state across multiple components?
Answer:
Context API is a built-in React feature for sharing data across the component tree without passing props down every level (called prop drilling).
When to use: Theme, language, authentication status, user data — anything needed by many components.
jsximport { createContext, useContext, useState } from 'react';

// Step 1: Create context
const ThemeContext = createContext();

// Step 2: Provide context value
function App() {
  const [theme, setTheme] = useState("light");

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      <Toolbar />
    </ThemeContext.Provider>
  );
}

// Step 3: Consume context anywhere in the tree
function Toolbar() {
  return <ThemedButton />;  // No need to pass theme as prop!
}

function ThemedButton() {
  const { theme, setTheme } = useContext(ThemeContext);  // Access directly

  return (
    <button
      style={{ background: theme === "light" ? "#fff" : "#333", color: theme === "light" ? "#000" : "#fff" }}
      onClick={() => setTheme(theme === "light" ? "dark" : "light")}
    >
      Toggle Theme (Current: {theme})
    </button>
  );
}

14. State Management (Redux, Redux-Toolkit, Recoil)
Q1: What is Redux, and why is it used in React applications? Explain the core concepts of actions, reducers, and the store.
Answer:
Redux is a predictable state management library for JavaScript apps. It centralizes all application state in a single store, making it easier to track, debug, and manage state across large apps.
Core concepts:
User Action → dispatch(action) → Reducer → New State → Store → UI Re-renders
jsx// 1. ACTION — plain object describing what happened
const incrementAction = { type: "INCREMENT" };
const addTodoAction = { type: "ADD_TODO", payload: "Buy groceries" };

// 2. REDUCER — pure function: (state, action) => newState
function counterReducer(state = { count: 0 }, action) {
  switch (action.type) {
    case "INCREMENT": return { count: state.count + 1 };
    case "DECREMENT": return { count: state.count - 1 };
    default: return state;
  }
}

// 3. STORE — holds the entire app state
import { createStore } from 'redux';
const store = createStore(counterReducer);

store.dispatch({ type: "INCREMENT" });
console.log(store.getState());  // { count: 1 }

jsx// Recoil — much simpler!
import { atom, useRecoilState, RecoilRoot } from 'recoil';

// atom = piece of state
const countAtom = atom({ key: 'count', default: 0 });

function Counter() {
  const [count, setCount] = useRecoilState(countAtom);
  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
    </div>
  );
}

function App() {
  return <RecoilRoot><Counter /></RecoilRoot>;
}
